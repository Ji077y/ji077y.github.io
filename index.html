解析过程涉及到一个关键的技术参数：TTL，Time to Live

TTL

TTl原理参考

TTL是IP协议包中的一个值，表示数据包在网络中的时间是否太长而应该被丢弃。有很多原因导致了一个包在一定时间内无法到达目的地。解决方案是在一定时间内丢弃这个数据包，然后发送一个报文给发送这个包的设备，由设备决定是否重新发送数据包。

TTL有缺省值，最初的设想是确定一个时间范围，超过此时间就把包丢弃

由于每经过一个路由器都至少要将TTL值减去1，所以TTL的值可以反映出这个包在被丢弃前最多能经过的路由器个数。

  • 当TTL计数达到0的时候，当前路由器就会将这个包丢弃并发送一个ICMP报文给最初的发送者

TTL字段的值可以帮助我们判断操作系统类型：

  • UNIX 及类 UNIX 操作系统 ICMP 回显应答的 TTL 字段值为 255
  • Compaq Tru64 5.0 ICMP 回显应答的 TTL 字段值为 64
  • 微软 Windows NT/2K操作系统 ICMP 回显应答的 TTL 字段值为 128
  • 微软 Windows 95 操作系统 ICMP 回显应答的 TTL 字段值为 32
  • 在有些情况下返回TTL的数值会较为特殊，这里就不展开阐述了
